# 🧭 LicenseHub（ライセンスハブ）概要

> **資格の「学び」と「キャリア」をつなぐ情報ハブ。**
> 登録販売者からスタートし、将来的には資格を通じて
> 「転職」「独立」「キャリアアップ」を比較・発見できるプラットフォームへ。

---

## 🎯 コンセプト

| 軸           | 内容                                                           |
| ------------ | -------------------------------------------------------------- |
| **主目的**   | 資格を「取る」だけでなく、「活かして働く」までをサポートする。 |
| **初期対象** | 登録販売者（tohan）                                            |
| **将来展開** | 簿記、FP、宅建、介護福祉士など主要資格に拡張                   |
| **更新方針** | AI・JSON・自動スクリプトによる半自動更新（試験日・合格率など） |
| **制作基盤** | Bloggrify + Nuxt Content + Markdown 構成（Git 管理）           |

---

## 🧩 コンセプト骨子（3 本柱）

| 軸                       | 目的                       | ページ領域                      |
| ------------------------ | -------------------------- | ------------------------------- |
| **1. 学ぶ（Learn）**     | 試験・教材・勉強法を知る   | `/exam`, `/materials`           |
| **2. 活かす（Work）**    | 資格でどんな仕事ができるか | `/work`                         |
| **3. 比べる（Compare）** | 他資格との違いや相場を知る | `/compare`, `/licenses/` トップ |

## 🏗 サイト構成（フェーズ別）

### フェーズ 1：登録販売者特化（現在）

```
/licenses/tohan/
 ├ exam/          # 試験概要・日程・難易度・申込情報など
 ├ materials/     # 教材・通信講座・独学法など
 ├ work/          # 年収・キャリア・求人・実務紹介
 └ articles/      # ニュース・体験談・お知らせ
```

### フェーズ 2：横展開（資格比較フェーズ）

```
/licenses/
 ├ tohan/         # 登録販売者
 ├ boki/          # 簿記
 ├ fp/            # FP
 ├ compare/       # 比較・診断コンテンツ
 │  ├ salary/     # 平均年収比較
 │  ├ difficulty/ # 難易度・合格率比較
 │  └ career/     # 活かせる職種比較
 └ articles/
```

### フェーズ 3：キャリア・転職導線拡張

```
/licenses/
 ├ [slug]/        # 各資格トップ（例：/licenses/tohan/）
 ├ compare/       # 横断比較
 ├ careers/       # 転職・実務・求人系特集
 └ guide/         # 学習・費用・期間などの総合ガイド
```

## 🧠 情報体験の方向性（UX 指針）

| 要素                              | 概要                                                            |
| --------------------------------- | --------------------------------------------------------------- |
| **読む → 触る体験へ**             | グラフ・比較 UI・診断フォームを通じて「操作して理解」できる構成 |
| **視覚的要約（Visual Literacy）** | 冒頭で 3 ポイントまとめを表示／チャート・要約カード活用         |
| **個人のリアル重視**              | 合格者・現場の声を混ぜ、AI では書けない文脈価値を付与           |
| **AI 要約対策**                   | 見出し構造・箇条書きを明確化し、AI 検索結果に引用される設計     |
| **再訪促進**                      | 「比較結果保存」「更新通知」など体験型コンテンツに発展予定      |

## 🧱 更新・AI 連携方針

| 項目                 | 実装方針                                                   |
| -------------------- | ---------------------------------------------------------- |
| **試験日・合格率等** | JSON にまとめ、ビルド時に AI が文章化                      |
| **教材・講座情報**   | A8.net 等からデータ取得＋ AI が要約生成                    |
| **記事更新**         | Markdown frontmatter を AI が自動更新（date, keywords 等） |
| **事実検証**         | 人間レビューを最後に実施（AI 誤情報対策）                  |
